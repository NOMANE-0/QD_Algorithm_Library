# 相机外参标定：理论与实践指南



相机外参的本质是描述**相机坐标系**与**世界坐标系**之间的三维空间刚体变换关系。在当前机器人的具体工况下，其核心目的是**精确标定相机坐标系与云台坐标系的空间相对位姿**。

外参由旋转参数（Roll, Pitch, Yaw，简称 rpy）和平移参数（X, Y, Z，简称 xyz）共同组成。在底层代码实现中，相机坐标系的零点，即物理对应于 `cv::solvePnP` 算法输出的平移向量 `tvec` 的绝对零点。

## 1. 相机外参的核心影响与风险链条



由于机械加工与装配不可避免地存在公差，相机物理上必定无法与云台完全平行对齐。这种物理偏差如果不加以软件补偿，会直接破坏系统的弹道解算。

* **核心痛点**：造成子弹实际落点在垂直方向上偏高或偏低。
* **置信度与影响权重**：在 RM 实际工况中，外参的 `pitch`（俯仰角）误差对最终命中率的影响**最大**。根据工程经验，其他两个角度（Roll 和 Yaw）的装配误差通常具有**高置信度**保持在 **0.5 deg 以内**，影响相对次要。
* **错误传播逻辑**：错误的外参 `pitch` $\rightarrow$ 导致目标从相机局部坐标系变换到世界全局坐标系后的高度 `z` 出现异常 $\rightarrow$ 进而引发弹道解算补偿的 `cmd_pitch` 数据异常 $\rightarrow$ 最终表现为子弹落点异常偏离预期。

## 2. 方案一：手动标定 (低资源环境备用方案)

> **批判性评估**：
> * **优势**：无需复杂的标定板与算法，对环境光照要求低，适合在极端受限环境下快速实施。
> * **劣势（风险）**：精度高度依赖操作者人眼观察的生理极限；受主观判断影响较大；对 `pitch` 等关键参数的标定成功率缺乏绝对的数学保障。

在环境条件受限时，只能以最低硬件条件完成外参标定。以下提供从平移和旋转两个维度的极简标定逻辑：

### 2.1 平移参数 (xyz) 的光心测定法



**数学推演**：假设相机的入射瞳在世界坐标系下的绝对位置为 $P_{pupil}$，空间内有共线的两点为 $X_{near}$ 和 $X_{far}$。当相机绕某物理轴进行旋转操作（旋转矩阵为 $R$）时，若该旋转中心严格重合于 $P_{pupil}$，则在旋转前后，入射瞳的绝对坐标始终保持 $P_{pupil}$ 不变。
此时，空间点 $X$ 在相机坐标系下的方向向量为 $v=R\cdot(X-P_{pupil})$。
对于原本在视野中重叠的两点（即满足共线线性关系 $X_{near}-P_{pupil}=k(X_{far}-P_{pupil})$），旋转后的方向向量 $v_{near}$ 和 $v_{far}$ 依然严格保持线性缩放关系。这在图像上的物理表现为：**两点坐标依然重合，不产生视差。**

**操作规程**：
1.  在相机视野中，刻意对齐一近一远两个静止物体（例如近处的窗框与远处的树木）。
2.  缓慢旋转相机本体。
3.  观察这两个物体之间是否发生相对位移（即视差现象）。
4.  前后移动相机位置并重复旋转测试，直到旋转时两者在画面中**绝对无相对位移**。

**结论**：此时相机的物理旋转轴即精确穿过了其光学中心（更严谨的物理表述为入射瞳 Entrance Pupil）。此法全程无需数值计算，最终精度取决于人眼的观察极限，理想状态下通常可达毫米级。

### 2.2 旋转参数 (rpy) 的闭环调节法

在 RM 实战工况下，优先保证标定好 `pitch` 即可达到极高的实战收益。
*(注：以下两种经验方法处于中/低置信度，单次测试难以提供 100% 标定成功的绝对保障，需反复验证)*

* **方法 A：基于斜率变化的动态调节**
    * **物理依据**：当目标物体相对于地面的物理高度固定时，控制其沿深度方向（前后）移动。理想情况下，相机识别该目标并将其坐标变换到世界坐标系后，输出的高度值 `z` 应当是一个不随距离变化的常数。
    * **操作逻辑**：以此作为闭环反馈信号，手动调节 `pitch` 参数，直到目标前后移动时，系统输出的 `z` 值无明显斜坡状波动变化。
    * **工具支持**：在 `rmvision2025` 框架中，已提供专用脚本 `calibrate_gimbal2camera_pitch.py` 与对应的功能开关 `enable_pitch_calibration` 以降低人工标定压力。开启后只需前后推拉移动目标，系统即可自动收集数据完成标定。

* **方法 B：基于 z_world 不变性的系统性调节**
    * 以下引用自 `rmvision` 开源库作者（陈君）的核心逻辑分享：

> 好多人问我 solvePnP 出来结果不准的问题，我做个我自己的心得分享：solvePnP 的输入参数有 objectPoints - 世界坐标系下的控制点的坐标 imagePoints - 在图像坐标系下对应的控制点的坐标 cameraMatrix - 相机的内参矩阵 distCoeffs - 相机的畸变系数，而输出的平移向量就是相机与装甲板的距离，而这个距离与上述四个参数都有关，不单是相机标定准就能解决的。
>
> 先讲相机内参问题，如果使用 Matlab 的工具箱，标定后能够获得相机与棋盘格的距离，如果该距离准确则可以排除相机内参的问题。多次标定后观察结果是否大致相同也能排除标定过程的不当。而装甲板处于图像中心时畸变影响一般不大，暂不考虑。
>
> 若标定正确，则 objectPoints 和 imagePoints 是大部分同学没考虑到的点。这两个参数是互相匹配的，若 objectPoints对应的是装甲板两个灯条上下边缘的两点，则输入的 imagePoints 也应是图像中装甲板两个灯条上下边缘的两点。以传统视觉为例，如果相机的曝光时间及预处理的阈值设置不当，那么识别器得到的点可能没有正确的匹配 objectPoints，导致 solvePnP 结果出现偏差。可以通过放大图像仔细观察 imagePoints 是否正确落在装甲板的灯条边缘来。
>
> 在保证输入参数都正确的情况下，采用 EPnP 的方法一般都能得到相对准确的距离。

## 3. 方案二：手眼标定 (Calibrate HandEye - 推荐方案)

> **批判性评估**：
> * **优势**：基于严密的代码推演与线性代数方程，系统客观；能够一次性解算高精度的完整外参（旋转与平移）。
> * **劣势（风险）**：前置条件苛刻；对硬件（需高精度标定板）、环境光照以及内参和 PnP 算法的先验精度要求极高；操作变量控制繁琐。

手眼标定在机器人学中分为“眼在手上 (Eye-in-Hand)”和“眼在手外 (Eye-to-Hand)”。根据当前云台相机的物理构型，本节**仅讨论眼在手上的情况**。



手眼标定在数学上通常被严密建模为经典的刚体变换方程：$$AX=XB$$。
*(注：此处不展开复杂的公式推理，核心在于理解求解顺序)*。在实际部署的工业标定算法（如 Tsai 算法等）中，底层计算逻辑**永远是先求解旋转矩阵，再代入求解平移向量**。
**关键结论**：这一数学特质决定了，手眼标定输出的**旋转参数通常是高准度的**，但平移参数因存在计算误差的累积传递，不一定完美。

**标定程序**：[OpenCV Calibration](https://gitee.com/slime0rimiru0/open-cv_-calibration)

### 3.1 标定规范与高置信度技巧

为确保 $AX=XB$ 方程组的良态解，必须严格控制以下操作变量：

* **运动解耦**：标定时标定板必须绝对静止，机器人底盘必须绝对静止，**仅允许操控云台旋转轴进行运动**。（注：尚未验证同时移动机器人底盘是否会引入破坏性的系统噪声）。
* **内参前置约束 (极高优先级)**：准确的相机内参是后续解算的绝对基石，务必将重投影误差压制在 **0.1 px** 级别。
* **硬件要求**：必须使用高精度、低反光率的专业标定板（例如氧化铝标定板）。
* **环境控制**：标定环境光照必须极佳，确保标定板的角点在画面中边缘分明无虚影。（经验反馈：下午柔和且充足的自然光下往往标定效果最好）。
* **稳态采样**：收集标定图像时，必须等云台完全停稳后再触发采集，严禁在运动中抓帧产生动态模糊。
* **数据多样性**：收集的标定图像序列中，相机相对标定板的位姿与角度变化跨度必须足够大，以保证方程组满秩。
* **PnP 精度**：前置的 `solvePnP` 算法解算精度必须高，否则无法获取准确的方程系数。
